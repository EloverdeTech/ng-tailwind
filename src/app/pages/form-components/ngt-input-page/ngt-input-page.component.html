Visual Inputs:

label

Input label.

placeholder

Input placeholder.

shining

When true, the component will display a shining mode.

loading

When true, the component will display a loading mode.

helpTitle

Component help title.

helpText

Determines if help text is enable.

innerLeftIcon

Internal icon displayed at start of the input.

innerRightIcon

Internal icon displayed at end of the input.


Behavior Inputs:

isDisabled

When true, the component will be disabled.

isReadonly

When true, the component will be readonly.

type

Determines the type of input expected by the component. Can be: text, email and password.

name

Name used by NgModel. Must be unique.

mask

Apply a mask to the component according to your need. Can be: cnpj-cpf, time, cellphone, plate, cep, decimal and integer.

focus

Focuses input automatically after view init.

allowClear

When true, the input can be clear by clicking in the clear icon of the component.

jit

When true, the ng model will be updated at typing time.


Validation Inputs:

isRequired

Determines whether the field is required.

uniqueResource

Used when you want to have only one record with a certain value.

The single record must be passed to this @Input() for validation to take place.

The validation will be done through the NgtHttpValidationService service that must be provided to the component.

minValue

Determines a minimum number value to be provided.

max

Determines the highest accepted value when using a integer mask.

maxlength

Defines the maximum number of characters accepted.

match

Used when the value of an entry must be identical to that of another input. For example, a password confirmation.

multipleOf

Only accepts values ​​that are multiples of the value provided for @Input().

multipleOf

Only accepts values ​​that are multiples of the value provided for @Input().


Methods:

setFocus

Focuses the input.

clearInput

Clears the input value.

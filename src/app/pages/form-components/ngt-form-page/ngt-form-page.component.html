Component used to manage and facilitate the control of other components in this form.

Note: This component needs the service NgtHttpFormService to be provided for it.


Examples:

FormState: {{ ngtForm.getFormState() }}

{{ codeExample }}

Inputs:

guessFormState

When true, the component will automatically identify the status of editing and creation.

message

Message shown when the form has a filling error.

routeIdentifier

The key for identify a edition route. The default is "id".

resource

Model used by the form to perform actions, like save.

customLayout

When true, you can choose where to display the form's validation message by placing the NgtFormValidationMessage component in the desired location inside the form.


Outputs:

onCreating

Event emitted when the form is in creation mode.

onEditing

Event emitted when the form is in edition mode.

onLoadingChange

Event emitted when the load attribute has change.

onShiningChange

Event emitted when the shining attribute has change.

setupComponent

Event emitted when the component is ready.


Methods:

isCreating

Returns true when the form state is "CREATING", false otherwise.

isEditing

Returns true when the form state is "EDITING", false otherwise.

isLoading

Returns true when the form is in loading mode, false otherwise.

isShining

Returns true when the form is in shining mode, false otherwise.

setShining

Setter for shining attribute. Must be a boolean value.

setLoading

Setter for loading attribute. Must be a boolean value.

setFormState

Setter for formState attribute. Must be a instance of NgtFormState.

Receives as a second parameter a boolean value that defines whether the change in the form's state should trigger the component's events related to that state.

getFormState

Returns the current formState.

saveResource

Saves the resource passed into resource attribute based on NgtHttpFormService provided for this component.

clearCachedRoute

Clear the cached route based on cached route reuse strategy.


Enums:

NgtFormState

The formState provided for this component must be one of those.

CREATING

Type used when you want to have a form for registering new data.

EDITING

Type used when you want to have a form for editing existing data.
